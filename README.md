# سامانه یکپارچه انتخاب واحد و زمان‌بندی امتحانات

## معرفی پروژه
این پروژه یک سامانه نرم‌افزاری برای تحلیل و مدیریت چارت درسی دانشگاهی و زمان‌بندی امتحانات است. هدف سیستم بررسی صحت پیش‌نیازهای دروس، پیشنهاد یک ترتیب مناسب برای اخذ دروس و همچنین تولید یک زمان‌بندی امتحانات بدون تداخل است.

در این سامانه روابط پیش‌نیازی دروس به صورت گراف جهت‌دار مدل‌سازی شده و تعارض‌های امتحانی به صورت گراف بدون‌جهت در نظر گرفته شده‌اند. سپس با استفاده از الگوریتم‌های کلاسیک گراف مانند جستجوی عمقی (DFS)، مرتب‌سازی توپولوژیک و رنگ‌آمیزی گراف تحلیل لازم انجام می‌شود.

رابط کاربری این سیستم با Next.js پیاده‌سازی شده و هسته الگوریتمی آن با JavaScript خالص نوشته شده است تا از کتابخانه‌های آماده برای پیاده‌سازی الگوریتم‌ها استفاده نشود.

---------------------------------------------------------------------

## تعریف مسئله

در سیستم‌های آموزشی دانشگاهی دو چالش مهم وجود دارد:

1. بررسی اعتبار چارت درسی از نظر پیش‌نیازها
2. زمان‌بندی امتحانات به گونه‌ای که تداخل برای دانشجویان ایجاد نشود

در صورتی که در روابط پیش‌نیازی یک چرخه وجود داشته باشد، برنامه درسی نامعتبر خواهد بود زیرا یک درس به طور غیرمستقیم پیش‌نیاز خودش خواهد شد.

از طرف دیگر در زمان‌بندی امتحانات باید اطمینان حاصل شود که دروس دارای دانشجوی مشترک در یک بازه امتحانی قرار نگیرند.

هدف این پروژه حل این دو مسئله با استفاده از الگوریتم‌های گراف است.

---------------------------------------------------------------------

## ورودی سیستم

ورودی سیستم یک فایل JSON است که شامل اطلاعات زیر می‌باشد:

- لیست دروس
- روابط پیش‌نیاز
- تعارض‌های امتحانی
- حداکثر تعداد درس در هر ترم

ساختار ورودی:

{
  "courses": [],
  "prerequisites": [],
  "conflicts": [],
  "maxCoursesPerSemester": 3
}

---------------------------------------------------------------------

## خروجی سیستم

پس از اجرای الگوریتم‌ها سیستم اطلاعات زیر را تولید می‌کند:

- بررسی وجود چرخه در گراف پیش‌نیاز
- ترتیب معتبر اخذ دروس
- برنامه پیشنهادی ترم‌ها
- زمان‌بندی امتحانات بدون تداخل
- دروس دارای بیشترین تعارض

---------------------------------------------------------------------

## مدل‌سازی گراف

### گراف پیش‌نیاز

پیش‌نیازهای درسی به صورت یک گراف جهت‌دار مدل می‌شوند.

G = (V , E)

V : مجموعه دروس  
E : مجموعه یال‌ها

اگر یال زیر وجود داشته باشد:

A → B

به این معناست که درس A پیش‌نیاز درس B است.

---------------------------------------------------------------------

### گراف تعارض امتحانی

برای زمان‌بندی امتحانات از گراف بدون‌جهت استفاده می‌شود.

در این گراف اگر بین دو درس یال وجود داشته باشد یعنی این دو درس نباید در یک زمان امتحان برگزار شوند.

---------------------------------------------------------------------

## الگوریتم‌های استفاده شده

### تشخیص چرخه با DFS

برای بررسی وجود چرخه در گراف پیش‌نیاز از الگوریتم Depth First Search استفاده شده است.

در این روش هر گره یکی از سه حالت زیر دارد:

0 : بازدید نشده  
1 : در حال بررسی  
2 : بررسی شده

اگر در حین DFS به گره‌ای با حالت 1 برسیم یعنی یک چرخه در گراف وجود دارد.

پیچیدگی زمانی:

O(V + E)

---------------------------------------------------------------------

### مرتب‌سازی توپولوژیک

اگر گراف فاقد چرخه باشد می‌توان از Topological Sort برای پیدا کردن ترتیب معتبر اخذ دروس استفاده کرد.

در این روش گره‌ها به ترتیبی مرتب می‌شوند که هر درس بعد از پیش‌نیازهای خود قرار گیرد.

پیچیدگی زمانی:

O(V + E)

---------------------------------------------------------------------

### برنامه‌ریزی ترم‌ها

پس از تعیین ترتیب دروس، یک الگوریتم حریصانه برای توزیع دروس در ترم‌ها استفاده می‌شود.

در این روش:

- دروس به ترتیب توپولوژیک بررسی می‌شوند
- در هر ترم تا سقف مجاز درس اضافه می‌شود
- پیش‌نیازها کنترل می‌شوند

---------------------------------------------------------------------

### رنگ‌آمیزی گراف برای زمان‌بندی امتحانات

برای جلوگیری از تداخل امتحانات از الگوریتم Graph Coloring استفاده شده است.

در این روش:

هر رنگ نشان‌دهنده یک زمان امتحان است.

دو گره‌ای که یال مشترک دارند نمی‌توانند رنگ یکسان داشته باشند.

پیچیدگی زمانی:

O(V + C)

C تعداد تعارض‌ها است.

---------------------------------------------------------------------

## تحلیل پیچیدگی

تشخیص چرخه:
O(V + E)

مرتب‌سازی توپولوژیک:
O(V + E)

ساخت گراف تعارض:
O(V + C)

رنگ‌آمیزی گراف:
O(V + C)

مرتب‌سازی دروس پرتعارض:
O(V log V)

پیچیدگی کل سیستم:

O(V + E + C + V log V)

V تعداد دروس  
E تعداد پیش‌نیازها  
C تعداد تعارض‌ها
